# Excel Image Server

Веб-сервис для вставки изображений товаров в Excel по артикулам.

Пользователь открывает сайт, прикрепляет `.xlsx` или `.xlsm`, выбирает лист и колонки, при необходимости выбирает локальные папки с изображениями, запускает обработку и автоматически получает файл с суффиксом `_with Images`.

Интерфейс сайта сделан на украинском языке. Этот README написан по-русски как техническая и эксплуатационная памятка. Все пути, домены и адреса ниже являются примерами.

## Возможности

- Вставляет первую найденную картинку артикула в указанную колонку Excel.
- Поддерживает `.xlsx` и `.xlsm`.
- Для `.xlsm` сохраняет макросы через `keep_vba=True`.
- Сохраняет формулы и существующее содержимое книги насколько это позволяет `openpyxl`.
- Старый `.xls` не поддерживается.
- Обрабатывает Excel-файлы строго по одному за раз.
- Позволяет нескольким пользователям открыть сайт одновременно, но сами обработки ставит в очередь.
- Показывает прогресс и позволяет отменить обработку.
- Автоматически скачивает результат после завершения.
- Не пишет оригинальный Excel-файл на диск в нормальном пути обработки.
- Хранит временные служебные файлы только в приватном `work_dir`.
- Публичный лог не содержит названий Excel-файлов, артикулов и пользовательских путей.

## Источники изображений

В интерфейсе есть три источника:

1. `Папка сервера` - изображения из серверного XML-индекса.
2. `Локальна папка 1` - папка, выбранная пользователем в браузере.
3. `Локальна папка 2` - вторая папка, выбранная пользователем в браузере.

По умолчанию серверная папка имеет приоритет `1`, локальные папки - `2` и `3`. Любой источник можно отключить через значение `Вимкнено`.

Если пользователь выбирает локальную папку, вся папка не отправляется на сервер сразу. Браузер локально сравнивает имена файлов с артикулами из Excel. На сервер отправляются только те изображения, которые реально подходят под строки выбранного Excel-файла.

## Правила поиска

Для серверного XML:

- берется первая картинка артикула;
- артикул из Excel и артикул из XML нормализуются;
- спецсимволы приводятся к единому виду;
- например, `_` и `-` могут совпасть после нормализации.

Для локальных папок:

- берется файл, у которого имя без расширения соответствует артикулу;
- совпадение должно быть полным;
- регистр не учитывается;
- `_` в имени файла сохраняет смысл;
- `0010.jpg` подходит к артикулу `0010`;
- `0010_1.jpg` не считается тем же самым, что `0010`;
- если подходит несколько файлов, берется первый найденный браузером.

Поддерживаемые форматы:

```text
.jpg, .jpeg, .png, .gif, .bmp, .tiff, .webp
```

## Вставка в Excel

- Картинка вставляется в маленький квадрат внутри ячейки.
- Ширина картинки подгоняется под ширину колонки.
- Высота строки подгоняется под фактическую высоту картинки.
- Слева и сверху остается минимальный отступ.
- Справа используется небольшой защитный запас, чтобы Excel не выпускал картинку за границу ячейки.
- Если включен фон ячейки, по умолчанию используется светло-серый цвет.
- Если включено `Не змінювати фон комірки`, заливка ячейки не трогается.

Сервис старается приблизиться к значению `Бажаний розмір результату, МБ`, но не превышает жесткий максимум `hard_max_output_mb`.

По умолчанию желаемый размер результата:

```text
20 МБ
```

Жесткий максимум результата:

```text
100 МБ
```

## Внешние ссылки Excel

Если книга содержит внешние ссылки через `ВПР`, `XLOOKUP` или похожие формулы, результат может открываться в Excel через восстановление.

Сервис открывает книгу с `keep_links=True`, то есть намеренно не удаляет внешние связи на этапе чтения. Но `openpyxl` не является полноценным Excel и не гарантирует идеальное сохранение всех внешних связей.

Практический риск:

- сохраненные значения ячеек обычно остаются;
- внешние связи могут быть повреждены или потеряны;
- Excel может показать окно восстановления при открытии результата.

Если файл критически зависит от внешних связей, сначала проверьте результат на копии.

## Конфиденциальность

Основная идея защиты: Excel-файл не должен задерживаться на диске сервера.

Как это работает:

- Excel не отправляется на сервер при простом выборе файла.
- Предварительный просмотр выполняется в браузере пользователя.
- На сервер Excel попадает только после нажатия кнопки обработки.
- Сервер читает Excel в оперативную память (`BytesIO`).
- В нормальном пути обработки оригинальный Excel не записывается в `work_dir`, `jobs`, `python_temp` или публичные папки.
- Результат тоже собирается в оперативной памяти (`BytesIO`).
- Временная папка процесса удаляется до отправки результата браузеру.
- После обработки буферы закрываются, ссылки на них удаляются.

Что важно понимать:

- При HTTP трафик не шифруется.
- Если сайт доступен без пароля, любой пользователь сети с доступом к адресу сервиса может открыть интерфейс.
- Оперативная память Python не затирается криптографически. Буферы закрываются и освобождаются штатными механизмами Python.
- Временные изображения удаляются best-effort: перед удалением файл перезаписывается нулями, но на SSD, сетевых дисках и файловых системах с журналированием это не является криптографически гарантированным уничтожением.
- Для более строгой защиты включайте шифрование диска Windows/BitLocker, ограничивайте доступ к порту и используйте HTTPS.

## Где могут быть временные файлы

Единственная рабочая папка задается параметром `work_dir` в `config.json`.

Внутри нее используются:

```text
<WORK_DIR>\jobs
<WORK_DIR>\python_temp
```

Во время обработки может появиться папка:

```text
<WORK_DIR>\jobs\process_<random_id>
```

Внутри нее временно могут быть только служебные изображения:

```text
local\
images\
```

Оригинальный Excel и итоговый Excel туда в нормальном пути обработки не пишутся.

Проверка временных файлов:

```powershell
Get-ChildItem "<WORK_DIR>\jobs" -Force
Get-ChildItem "<WORK_DIR>\python_temp" -Force
```

Во время активной обработки одна папка `process_*` может быть видна. После завершения, ошибки или отмены она должна исчезнуть.

Если осталась пустая папка `process_*`, сервис при следующей очистке удалит ее. Удалить вручную можно так:

```powershell
Remove-Item -LiteralPath "<WORK_DIR>\jobs\process_xxx" -Recurse -Force
```

## Публичные папки

Папки, которые отдает Caddy или другой статический сервер, должны быть отделены от `work_dir`.

Пример публичного лога:

```text
http://example.internal/static/xml/excel_image_server.log
```

Пример публичного XML:

```text
http://example.internal/static/xml/images_export.xml
```

Важно:

- `work_dir` не должен находиться в папке, которую отдает Caddy;
- `work_dir` не должен быть внутри публичного дерева файлов;
- XML и фото можно держать публично, если они не считаются секретными;
- Excel-файлы пользователей нельзя класть в публичные папки.

Код дополнительно проверяет конфиг и не дает запустить сервис, если `work_dir` находится внутри публичных XML/log/images папок или внутри общего публичного дерева.

## HTTP и HTTPS

Сейчас сервис рассчитан на HTTP внутри доверенной локальной или корпоративной сети.

При HTTP:

- данные передаются без шифрования;
- Excel можно перехватить внутри сети при наличии доступа к трафику;
- браузер и сервер не проверяют друг друга криптографически.

Если таблицы действительно чувствительные, оптимальный вариант:

1. держать `work_dir` приватным и локальным;
2. включить шифрование диска на сервере;
3. ограничить доступ к порту фаерволом или VPN;
4. включить HTTPS через reverse proxy;
5. при необходимости добавить авторизацию.

## Конфигурация

Основной конфиг лежит рядом с приложением:

```text
<INSTALL_DIR>\config.json
```

Пример без реальных путей и адресов:

```json
{
  "host": "0.0.0.0",
  "port": 8091,
  "xml_url": "http://example.internal/static/xml/images_export.xml",
  "images_dir": "C:\\ExampleData\\ProductPhotos",
  "images_base_url": "http://example.internal/static/foto/",
  "work_dir": "C:\\ExamplePrivate\\ExcelImageServer\\work",
  "public_log_dir": "C:\\ExamplePublic\\xml",
  "public_log_filename": "excel_image_server.log",
  "max_log_lines": 2000,
  "max_upload_mb": 100,
  "max_local_images_upload_mb": 500,
  "hard_max_output_mb": 100,
  "default_desired_output_mb": 20,
  "default_start_row": 2,
  "default_article_column": "A",
  "default_image_column": "B",
  "default_cell_background_color": "D9D9D9",
  "image_padding_px": 1,
  "image_width_guard_px": 6,
  "download_timeout_seconds": 20
}
```

Параметры:

- `host` - адрес прослушивания. Для доступа из сети обычно `0.0.0.0`.
- `port` - порт сайта.
- `xml_path` - локальный путь к XML-индексу изображений. Необязателен, если указан `xml_url`.
- `xml_url` - URL XML-индекса. Необязателен, если указан `xml_path`.
- `images_dir` - локальная серверная папка с фото.
- `images_base_url` - URL-корень фото, который указан в XML.
- `work_dir` - приватная рабочая папка временных файлов.
- `public_log_dir` - папка публичного лога.
- `public_log_filename` - имя публичного лога.
- `public_log_url` - опциональная готовая ссылка на лог.
- `max_log_lines` - максимальная длина публичного лога. По умолчанию 2000 строк.
- `max_upload_mb` - максимальный размер Excel-файла.
- `max_local_images_upload_mb` - максимальный общий размер локальных картинок, которые браузер отправит на сервер.
- `hard_max_output_mb` - жесткий максимум результата.
- `default_desired_output_mb` - желаемый размер результата по умолчанию.
- `default_start_row` - начальная строка по умолчанию.
- `default_article_column` - колонка артикула по умолчанию.
- `default_image_column` - колонка картинки по умолчанию.
- `default_cell_background_color` - фон ячейки с картинкой по умолчанию.
- `image_padding_px` - минимальный отступ картинки сверху и слева.
- `image_width_guard_px` - защитный запас справа.
- `download_timeout_seconds` - timeout загрузки изображения по URL.

Windows-пути в JSON нужно писать с двойными обратными слешами:

```json
"work_dir": "C:\\ExamplePrivate\\ExcelImageServer\\work"
```

Нельзя писать так:

```json
"work_dir": "C:\ExamplePrivate\ExcelImageServer\work"
```

## Автообновление

Супервизор проверяет Git с заданным интервалом.

Если есть обновление:

1. супервизор спрашивает у сервера `/status`;
2. если сервер занят обработкой Excel, обновление откладывается;
3. когда сервер свободен, супервизор останавливает worker;
4. подтягивает актуальный код из Git;
5. переустанавливает зависимости;
6. перезапускает сервис.

Автообновление не должно прерывать текущую обработку файла.

Если `/status` недоступен, супервизор считает это риском и откладывает обновление, а не убивает процесс вслепую.

## Логи

Публичный лог находится там, куда указывает:

```json
"public_log_dir": "...",
"public_log_filename": "excel_image_server.log"
```

В приложении нет отдельного публичного `/log`. Лог должен открываться через Caddy или другой статический сервер.

Публичный лог нужен только для понимания, что сервис жив и работает. Он не является местом хранения пользовательских данных.

Лог сохраняет последние `max_log_lines` строк. По умолчанию это 2000 строк. Когда строк становится больше, старые строки удаляются, новые продолжают дописываться. При перезапуске сервиса лог не очищается.

## Что делать, если остались временные файлы

1. Убедиться, что обработка сейчас не идет:

```text
http://example.internal:8091/status
```

2. Проверить `busy`. Если `busy: false`, можно удалить пустые или старые папки:

```powershell
Get-ChildItem "<WORK_DIR>\jobs" -Directory -Force
Remove-Item -LiteralPath "<WORK_DIR>\jobs\process_xxx" -Recurse -Force
```

3. Если папка не удаляется, возможно, файл еще держит процесс. Тогда сначала остановить сервис:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "<INSTALL_DIR>\scripts\stop_tasks.ps1"
```

4. После остановки повторить удаление.

## Рекомендованные меры безопасности

Минимальный набор:

- держать сервис только во внутренней сети;
- закрыть порт снаружи на роутере или фаерволе;
- не публиковать порт приложения в интернет;
- держать `work_dir` на локальном диске сервера;
- не размещать `work_dir` внутри публичных папок;
- включить BitLocker или другое шифрование диска;
- ограничить права на `<INSTALL_DIR>` и `<WORK_DIR>`;
- регулярно проверять `<WORK_DIR>\jobs`;
- не писать в лог названия файлов, артикулы и пользовательские пути;
- делать тест на копии, если Excel содержит внешние связи.

Для более строгой модели:

- добавить HTTPS;
- добавить авторизацию;
- ограничить доступ по IP или VPN;
- запускать задачу под отдельным Windows-пользователем с минимальными правами;
- дать этому пользователю `Modify` только на `work_dir` и папку лога, `Read` на фото/XML;
- включить аудит доступа к рабочей папке.
